## 🎯 LOS 4 CSV ESENCIALES

### 1️⃣ pacientes.csv
```
Estado actual de pacientes (82 registros)
├─ id_paciente: Identificador único
├─ edad: Edad del paciente
├─ genero: F/M
├─ nivel_urgencia: 1-5 (crítico a leve)
├─ sintomas: Descripción de síntomas
├─ departamento: Dpto asignado
└─ estado: En_espera, En_atencion, Alta, Transferido
```

### 2️⃣ recursos.csv
```
Disponibilidad de recursos (5 departamentos)
├─ departamento: Nombre del dpto
├─ total: Camas/recursos totales
└─ ocupados: Camas/recursos en uso
```

### 3️⃣ personal.csv
```
Personal hospitalario (55 empleados)
├─ id_empleado: ID único
├─ rol: Doctor/Enfermero
├─ especialidad: Especialidad médica
├─ departamento_1: Dpto primario
├─ departamento_2: Dpto alternativo
└─ estado: Disponible/Descanso
```

### 4️⃣ metricas.csv
```
Indicadores consolidados (1 resumen actual)
├─ pacientes_activos: Cantidad actual
├─ ocupacion_pct: Porcentaje de ocupación
└─ casos_criticos_pct: % de casos urgentes (1-2)
```

## 🔄 FLUJO DE DATOS SIMPLIFICADO

```
CSV Files (Estado Actual del Hospital)
    ↓
HospitalStateManager (Carga)
    ├─ load_patients() → pacientes.csv
    ├─ load_resources() → recursos.csv
    ├─ load_staff() → personal.csv
    └─ load_historic_metrics() → metricas.csv
    ↓
FastAPI Endpoints (/api/*)
    ├─ /api/patients
    ├─ /api/departments
    ├─ /api/resources
    ├─ /api/staff
    └─ /api/metrics
    ↓
Frontend Components
    ├─ Dashboard
    ├─ PatientsTable
    ├─ HospitalMetrics
    └─ EventTimeline
    ↓
🎨 Visualización en Navegador
```

## 🚀 PRÓXIMOS PASOS

1. **Iniciar Backend**
   ```bash
   cd hospital-SMA
   .venv\Scripts\Activate
   python -m uvicorn backend.main:app --reload --host 0.0.0.0 --port 8000
   ```

2. **Iniciar Frontend**
   ```bash
   cd frontend
   npm run dev -- --host localhost --port 5174
   ```

3. **Acceder al Dashboard**
   ```
   http://localhost:5174
   ```

4. **Verificar CSV regularmente**
   ```bash
   python verify_csv.py
   ```
